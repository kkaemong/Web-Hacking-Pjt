이 문서는 **한국정보보호센터 홈페이지** 및 관련 정보 자산에 대한 취약점 진단 및 평가 결과를 요약한 README 파일입니다.

## I. 개요

### 1.1 모의해킹 정의

본 모의해킹은 한국정보보호센터 홈페이지와 관련된 모든 정보 자산에 대한 취약점을 도출/분석하고 대책을 수립하기 위해 수행되었습니다. 해커와 동일한 환경과 조건, Hacking Skill을 가지고 모의적인 침투에 의해 이루어졌으며, 발견된 취약점에 대해서는 사전적인 예방을 통한 효과를 발생시키는 데 목적이 있습니다.

### 1.2 모의해킹 수행 과정

본 모의해킹은 다음 단계별로 진행되었습니다: 

| 수행 단계 | 설명 |
| :--- | :--- |
| **정보 수집 (Information Gathering)** | 대상 서버/네트워크/서비스에 대한 불필요한 서비스 접근 가능성, 외부에서 파악할 수 있는 정보 등을 수집하는 단계. |
| **취약점 수집 (Vulnerability Assessment)** | 각 네트워크 구간별 적합한 취약점 스캔 도구를 이용해 발생할 수 있는 취약점에 대한 정보를 수집하는 단계 (단, 네트워크 장비/서비스에 장애를 유발할 수 있는 경우는 제외). |
| **침투 단계 (Gaining Access)** | 취약점 수집을 통해 획득한 정보를 기반으로 수동점검(Manual)을 통해 내부 시스템까지 침투할 가능성이 있는지 시나리오 기반으로 확인. |
| **상세 분석 (Maintaining Access)** | 취약점이 도출되었을 경우 공격에 의해 보안 위험이 시스템 및 비즈니스 측면에서 어느 정도의 영향을 줄 수 있는지 분석하는 단계. |
| **보고서 작성 (Report)** | 도출된 취약점에 대한 총평/영향도/상세분석/보안가이드가 포함된 보고서를 작성하는 단계. |

## III. 취약점 진단 항목 및 평가 기준

### 3.1 진단 항목

OWASP TOP 10, CWE/SANS TOP25, 주요정보통신기반 기술적 취약점 분석 평가 상세가이드의 웹 진단 항목을 준용하여 취약점을 선정하고 수행했습니다.

| 순번 | 분류 | 진단 항목 |
| :--- | :--- | :--- |
| 1 | 불충분한 인가 | 민감한 기능 또는 기능에 접근 및 수정 시 통제 여부 점검 |
| 2 | 불충분한 인증 | 중요 페이지 접근 시 추가 인증 요구 여부 점검 |
| 3 | 약한 문자열 강도 | 웹 페이지 내 로그인 폼 등에 의한 강도의 문자열 사용 여부 점검 |
| 4 | 버퍼 오버플로우 | 사용자가 입력한 파라미터 값의 문자열 길이 제한 확인 |
| 5 | SQL Injection | 웹 페이지 내 SQL 인젝션 취약점 존재 여부 점검 |
| 6 | 크로스 사이트 스크립팅(XSS) | 웹 페이지 내 XSS 취약점 존재 여부 점검 |
| 7 | 디렉터리 인덱싱 | 디렉터리 인덱싱 여부 |
| 8 | 파일 다운로드 취약점 | 웹 사이트에서 파일 다운로드 시 허용된 경로 외 다른 경로의 파일 접근이 가능한지 여부 점검 |
| 9 | 불충분한 세션 만료 | 세션의 만료 기간 설정 여부를 점검 |
| 10 | 자동화 공격 | 웹 애플리케이션의 특정 프로세스(로그인, 게시글 등록 등)에 대한 반복적인 요청 시 통제 여부 확인 |

### 3.2.1 취약점 위험도 평가 기준

| 위험도 | 설명 |
| :--- | :--- |
| **매우 높음 (VH)** | 외부 노출된 서비스를 통해 내부 네트워크 대역에 침투가 가능하여 개인 정보 및 사내 주요 정보들이 외부에 노출될 위험성이 높음. 시스템 관리자 권한 획득으로 인한 시스템에 심각한 영향을 발생시켜 비즈니스적인 큰 피해를 줄 위험성이 높음. |
| **높음 (H)** | 시스템에 일부 심각한 영향이 발생하나 2차적인 피해 발생이 이루어질 가능성이 적음. |
| **중간 (M)** | 외부 노출된 서비스를 통해 시스템 정보 및 중요 정보가 **제한된 환경에서만** 노출됨. |
| **낮음 (L)** | 추가적인 공격 가능성을 줄 수 있는 정보 노출. |
| **매우 낮음 (VL)** | 서비스에 영향을 미치지 않고 일부 불필요한 정보가 노출됨. |

## IV. 진단 결과

### 4.1 진단 결과 요약

#### A. 로그인/회원 정보 관련 취약점

| 페이지 | 취약점 | 요약 |
| :--- | :--- | :--- |
| **로그인** | 불충분한 인증 | 로그인 후 중요 정보 페이지(개인 정보 변경 등)에 추가적인 2차 인증 또는 인증이 필요한 곳에 추가적인 검증을 하지 않음. |
| **로그인** | 약한 문자열 강도 | 계정 정지/금지에 대한 횟수 제한이 설정되어 있지 않아 비밀번호를 3~5회 정도 틀렸음에도 제한이 걸리지 않음. |
| **로그인** | 버퍼 오버플로우 | 로그인 화면에 인가된 계정 말고 대량의 문자열을 입력했을 때 로그인이 성공됨. |
| **회원 정보** | 불충분한 인가 | 접근제가 필요한 중요 페이지에 서버 측 통제 수단이 구현되지 않아 Burp Suite로 다른 계정 ID의 비밀번호가 바뀌게 됨. |

#### B. 커뮤니티 및 문의 게시판 관련 취약점

| 영역 | 취약점 | 설명 |
| :--- | :--- | :--- |
| **문의** | SQL Injection | SQL 쿼리문을 이용해 게시글 수정 시 SQL 쿼리 입력에 대한 검증이 이루어지지 않는 것을 발견. |
| **커뮤니티** | 디렉터리 인덱싱 | Burp Suite를 이용해 `/core` 경로를 발견하고 이를 통해 관리자 사이트를 발견. |
| **커뮤니티** | 파일 다운로드 취약점 | Burp Suite를 이용해 파일 다운로드 시 파일의 저장 경로를 알아내어 관리자 사이트 경로를 알아낼 수 있음. |
| **커뮤니티** | SQL Injection | SQLMAP을 사용하여 관리자 ID, PW를 알아냄. |
| **나의 강의실** | 크로스 사이트 스크립팅(XSS) | 검색창에서 `<script>alert(1)</script>`를 검색하여 경고창을 띄움. |
| **나의 강의실** | 불충분한 세션 만료 | 세션 만료 시간을 정하지 않아 2~3시간 이후에도 세션이 파기되지 않고 사용 가능. |
| **나의 강의실** | 자동화 공격 | 게시글 등록 또는 재로그인 시도 시 아무런 제한이 없는 것을 확인. |
